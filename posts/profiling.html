Code Profiling in C

22 December 2018

During the last month I had to work on a piece of code written in C. Besides adding some features, I wanted to optimize the code. I started googling for write better code c or make code faster c and found this nice article. I am but a physicist with limited knowledge in programming, so I thought that this article will help make the code even faster.

The big problem I did not see until this week is the following: How do I optimize code, when I don’t even know where there is most potential for optimization? That’s when I learned of Profiling.

Profiling using GPROF

I looked up profiling in c and found this easy way to do it. Just slap the flag -pg at the end of your compilation like

gcc -Wall -pg main.c -o profile

and execute the code once. Now a new file called gmon.out has been created. To produce the analysis of the code one then types

gprof profile gmon.out > profile.txt

and voila, we have the analysis of our code in the file profile.txt.

Understanding the output

GPROF spits out a so called flat profile which basically tells you how long the program executed each function. It also gives a call graph that depicts which function has been called by others and how often etc. For more details see this article or this one.
