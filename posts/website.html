Easy Website

9 December 2018

I’ve tried a lot of stuff to get an easy website, but this way seems the best.

I write the articles in markdown and then use pandoc to compile them into html pages using the CSS stylesheet I want.

Folder structure

The directory looks pretty much like this:

Website
├── blog.md
├── index.md
├── Makefile
├── posts
│   ├── quantfric.md
│   └── website.md
└── public
    ├── blog.html
    ├── css
    │   ├── blog.css
    │   └── index.css
    ├── img
    │   ├── christoph.png
    ├── index.html
    ├── js
    │   └── quantfric.js
    └── posts
        ├── quantfric.html
        └── website.html

The files index.html and blog.html are the startpage of the website and of the blog respectively.

In the folder posts I have all the markdown files that generate the html files.

Finally the public folder is the website itself as it lays on the server. Within it there are the CSS stylesheets, javascripts for some animations I did, pictures and other files and of course the relevant html files.

How it’s build

The website is build using the Makefile

# Makefile for Website

# directories
PUBDIR=public
POSTDIR=posts
CSSDIR=css

# other options
MATHOPT=mathjax
HIGHOPT=pygments

# get all posts
POSTS=$(wildcard $(POSTDIR)/*.md)
POSTSHTML=$(POSTS:.md=.html)

all: index.html blog.html $(POSTSHTML)

index.html: index.md
    pandoc -s $< -c $(CSSDIR)/index.css -o $(PUBDIR)/$@

blog.html: blog.md
    pandoc -s $< -c $(CSSDIR)/blog.css -o $(PUBDIR)/$@

# for posts
%.html: %.md
    pandoc -s --$(MATHOPT) --highlight-style $(HIGHOPT) $< -c ../$(CSSDIR)/blog.css -o $(PUBDIR)/$@

I have separate commands for the index, blog and all the posts. To create the html to a post called test one would type make posts/test.html into the console.

However, I usually just type make into the console to make the whole site, which for now does not even take a whole second.

As you can see the formulas are rendered with mathjax and the code highlighting is performed with pygments.
